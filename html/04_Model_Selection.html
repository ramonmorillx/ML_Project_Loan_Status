Undersampling, Oversampling y Modelado

Autor: Ramn Morillo Barrera

Dataset: Application data

En este notebook se llevar a cabo el tratamiento ltimo a los datos antes de la implementacin y anlisis de los diferentes modelos de ML. Realizaremos Oversampling y Undersampling con el objetivo de balancear nuestra variable objetivo y ver como se comportan los diferentes modelos de ML que implementaremos durante el entrenamiento. Implantaremos estos modelos a nuestros datos anteriormente estudiados, limpios y preprocesados. Estamos ante un problema de clasificacin, es por ello que intentaremos predecir la clase de nuestra variable objetivo. Se probarn varios modelos de ML estudiados:

  • Modelo Base
  • Modelo dummy
  • Modelo GLM
  • Modelo SVM
  • Modelo Random Forest
  • Modelo XGBoost
  • Modelo LightGBM
  • Modelo GradienBoosting
  • Modelo AdaBoost
  • Modelo Naive Bayes

Estos modelos se aplicarn con sus configuraciones por defecto, posteriormente se seleccionar la configuracin de datos que funcione mejor con nuestro modelo para terminar por buscar los hiperparmetros que mejores resultados obtengan. Para todos estos procesos de comparacin de modelos se utilizar un set de validacin, que surgir de volver a dividir en 70-30 al set de entrenamiento.

Train-Validation split

Antes de aplicar los modelos, separ nuestro conjunto de train en train-validation, con el objetivo de tener nuestro dataset de test intacto. Ya que con el train y validacin podremos hacer diversas pruebas y calcular diversas mtricas sin necesidad de utilizar nuestro dataset de test, mantenindolo intacto.

  • Con el dataset de entrenamiento entrenaremos nuestros modelos.
  • Con el dataset de validacin ajustaremos hiperparmetros y evaluaremos el desempeo del modelo durante el proceso de entrenamiento. Nos permitir detectar sobreajuste (overfitting) al comprobar cmo se desempea el modelo con datos no vistos durante el entrenamiento.
  • El dataset de test lo tendremos reservado para evaluar el desempeo final del modelo despus del entrenamiento y ajuste. As refleja cmo el modelo generaliza en datos completamente nuevos, simulando el desempeo en produccin.

Oversampling y Undersampling

He investigado en internet y he leido sobre una tcnica de rebalanceo que me parece interesante, es una combinacin proporcional de undersampling y oversampling. He decidido probarlo mediante la creacin de un Pipeline en donde se aplican secuencialmente las tcnicas.

Es una estrategia mixta, es decir, reducir la clase mayoritaria de nuestra variable objetivo y aumentar la clase minoritaria. Mi objetivo es realizar esta tcnica dejando un nmero de registros parecido al de nuestro dataset de train original. Esto har que los modelos tengan el mismo nmero de registros para ser entrenados y que solo difiera la proporcin de nuestra variable objetivo.

A continuacin realizar este proceso y comentar el porqu he decidido decantarme por esta tcnica en vez de utilizar las tcnicas originales, ya que dicha tcnica tiene sus claras ventajas.

In[7]:
# Under and Over Sampling applied to the training set
over = SMOTE(sampling_strategy=0.1, random_state=seed)
under = RandomUnderSampler(sampling_strategy=0.2, random_state=seed)

# Create pipeline
steps = [('o', over), ('u', under)]
pipeline = Pipeline(steps=steps)

# Apply resampling
X_train_resampled, y_train_resampled = pipeline.fit_resample(X_train_orig, y_train_orig)

Modelado

En esta seccin se analizarn los diferentes modelos seleccionados junto con los 2 datasets disponibles. El objetivo es finalizar el notebook con la eleccin del modelo ms adecuado, del dataset que mejor se adapte a este modelo y de los hiperparmetros que optimicen los resultados deseados. Para alcanzar este objetivo, se seguirn tres etapas principales:

  1. Comparacin de modelos: A travs de un pipeline, se evaluarn todos los modelos utilizando sus hiperparmetros por defecto, sin ajustes previos, con el fin de identificar cul ofrece el mejor desempeo inicial.

  2. Comparacin de datasets: Una vez seleccionado el modelo, se proceder a analizar cul de los conjuntos de datos logra las mtricas ms satisfactorias.

  3. Optimizacin de hiperparmetros: Finalmente, se realizar una bsqueda de los hiperparmetros ptimos para el modelo seleccionado utilizando el dataset elegido, empleando un enfoque de Validacin Cruzada.

A continuacin, se proceder a evaluar el resto de los modelos mediante un pipeline, asegurando una comparacin estructurada y consistente:

  • XGBoost Classifier: Un modelo destacado dentro de los algoritmos de boosting, ampliamente reconocido por su efectividad en problemas complejos de machine learning.
  • LightGBM Classifier: Otro modelo de boosting y uno de mis favoritos, gracias a su gran eficiencia y optimizacin en tiempos de procesamiento.
  • AdaBoost Classifier: Un modelo destacado dentro de los algoritmos de boosting, conocido por combinar mltiples clasificadores dbiles para lograr un rendimiento robusto en problemas de machine learning.
  • GradientBoosting Classifier: Un modelo sobresaliente en la familia de algoritmos de boosting, diseado para optimizar el rendimiento mediante la minimizacin progresiva del error en problemas complejos de clasificacin.
