Dataset: Application data

Sobre el dataset:

Se trata de una base de datos de registros de prstamos. Cada registro representa a un cliente al que se le ha concedido un prstamo, en los diferentes registros se enumeran las diferentes caractersticas de los clientes, las cuales vamos a analizar posteriormente.

Objetivos de la prctica

Esta prctica tiene como objetivo aplicar tcnicas de Anlisis Exploratorio de Datos (EDA) para comprender y analizar los patrones presentes en un conjunto de datos relacionado con solicitudes de prstamos. Supongamos que trabajamos para un banco que se especializa en otorgar varios tipos de prstamos a clientes. Tendremos que usar EDA para analizar los patrones presentes en los datos, asegurando que los solicitantes capaces de pagar el prstamo no sean rechazados. Se identificarn patrones que indiquen si un cliente puede tener dificultades para devolver el prstamo, lo que puede ser utilizado para tomar acciones.

Pasos a seguir

  1. Anlisis inicial de los datos y preprocesamiento inicial:

    • Iniciaremos explorando el conjunto de datos con el que trabajaremos, realizando un anlisis descriptivo de las variables y las instancias que lo componen. Este anlisis incluye examinar la distribucin de los datos y observar relaciones entre las distintas variables, prestando especial atencin a la variable objetivo (Target). A partir de este estudio, podremos extraer conclusiones sobre la estructura del conjunto de datos y los principales patrones que surgen en un anlisis exploratorio inicial.

  2. Correlaciones, tratamiento de missing y outliers:

    • En este paso, investigamos las correlaciones entre las variables, identificando relaciones significativas que podran influir en el modelo. Tambin se abordar el tratamiento de valores faltantes y de los outliers. Determinaremos cmo estos pueden afectar el anlisis y definiremos estrategias para su tratamiento o eliminacin en caso necesario.

  3. Tratamiento de variables categricas: encoding:

    • Las variables categricas sern codificadas para que puedan ser utilizadas en el modelo. Implementaremos tcnicas de encoding como One-Hot Encoding o Label Encoding, dependiendo de la naturaleza de cada variable, de manera que se optimice la compatibilidad de los datos con los algoritmos que se emplearn.

  4. Aplicacin de algoritmos:

    • Una vez preprocesados los datos, aplicaremos diferentes algoritmos predictivos. Esto implica realizar todos los procedimientos necesarios de seleccin de variables y preprocesamiento para asegurar que los datos sean compatibles con los modelos elegidos. Evaluaremos y compararemos el rendimiento de cada algoritmo, observando cul se ajusta mejor a nuestros datos y comprendiendo las razones detrs de su desempeo. Tambin abordaremos tcnicas para tratar datos desbalanceados y verificaremos la efectividad de su aplicacin en este contexto. Adems, se implementarn tcnicas de mejora de modelos, como la validacin cruzada (Cross Validation) y la optimizacin de hiperparmetros (Hyperparameters Tuning).

  5. Evaluacin con la muestra de test:

    • La evaluacin final se realizar con el conjunto de datos de prueba, aplicando el/los modelos seleccionados y midiendo su rendimiento en trminos de las mtricas elegidas. Esta evaluacin permitir sacar conclusiones sobre la efectividad del modelo en datos nuevos, asegurando que el modelo es robusto y confiable en su prediccin.

  6. Explicabilidad:

    • Finalmente, aadiremos una capa de interpretabilidad mediante el anlisis del funcionamiento interno del modelo. Usaremos mtodos especficos de explicabilidad para identificar las variables ms y menos importantes y comprender cmo influye cada variable en las decisiones del modelo. Este anlisis nos ayudar a entender de forma ms clara y transparente el proceso de decisin del modelo y los factores clave que impactan en sus predicciones.

Caso de negocio:

En la aplicacin de un modelo para predecir si los clientes devolvern un prstamo bancario, es crucial considerar el desbalance de datos, los objetivos especficos de negocio y la capacidad de la institucin para gestionar el riesgo de impagos. Estos factores son determinantes tanto en la fase de anlisis como en el modelado y evaluacin del modelo.

El objetivo es mejorar un modelo inicial basado en reglas simples con un modelo de machine learning capaz de detectar patrones complejos en los datos que indiquen la probabilidad de impago, algo que un sistema de reglas difcilmente podra captar. Esto permitir al banco no solo identificar clientes en riesgo, sino tambin anticiparse a patrones de incumplimiento menos evidentes.

Como no tenemos conocimiento del momento en el que se va a ejecutar el modelo cuando el cliente va a solicitar el prstamo, voy a suponer que el modelo se ejecutar en el momento en el que el cliente va a solicitar un prstamo, por tanto, observando las variables que se van a analizar no se encuentra ninguna variable a futuro. Trabajaremos con todas las variables que nos proporcionen informacin hasta el feature processing.

Para evaluar el rendimiento del modelo, se priorizarn mtricas como el F1 y F2 Score, con un enfoque particular en la sensibilidad (recall) para identificar la mayor cantidad posible de clientes en riesgo de incumplir el pago. Aunque este enfoque puede aumentar los falsos positivos (casos en los que se prev riesgo pero el cliente paga puntualmente), asumimos que la institucin tiene recursos para gestionar estos casos y que el beneficio de una deteccin temprana justifica este enfoque. Si se dispusiera de informacin de costes especficos, una matriz de coste permitira ajustar mejor el balance entre sensibilidad y precisin.

Este tipo de modelos suele enfrentarse a un desbalance de clases, con menos casos de incumplimiento que de cumplimiento, lo cual requiere tcnicas especficas para optimizar tanto el anlisis como el rendimiento del modelo. Adems, en el sector financiero es clave garantizar la explicabilidad del modelo. Se dedicar un anlisis especial para comprender la influencia de cada variable y explicar las decisiones del modelo, facilitando el cumplimiento con las normativas de transparencia y ayudando a comprender mejor los factores que impulsan el riesgo de impago.

Este enfoque integral permitir no solo prever posibles impagos con anticipacin, sino tambin comprender las razones subyacentes, alinendose con los objetivos y desafos de negocio en cada etapa del proyecto.
